# Docs Site

## What is included

The repository now includes:

- `mkdocs.yml`
- `requirements-docs.txt`
- `docs/` content pages
- `.github/workflows/docs.yml`

## Local preview

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-docs.txt
mkdocs serve
```

Then open the local MkDocs address shown in the terminal.

## Production deployment with GitHub Pages

The workflow in `.github/workflows/docs.yml` builds and deploys the site.

Required repository setting:

- In GitHub repository settings, enable **Pages** and choose **GitHub Actions** as the source.

After that:

- pushes to `main` will build the docs site
- manual deployment is also available via `workflow_dispatch`

## Why MkDocs was chosen

MkDocs is a good fit here because:

- the repo already centers around Markdown and config files
- contributors can write docs without a frontend toolchain
- local preview is simple
- GitHub Pages deployment is straightforward
- the documentation can stay close to the code and evolve with it

## Future upgrades

If the docs site grows further, reasonable next steps are:

- add autogenerated API/reference pages for selected modules
